# 文字游戏引擎 - Android App 开发文档

## 一、项目概述

### 1.1 项目简介
本文档描述一款基于AI的分布式文字冒险游戏开发框架。该游戏允许用户自定义游戏世界观、角色系统、状态机制，并提供持久化的数据存储能力。

### 1.2 核心特性
- 用户可自定义游戏背景、角色状态类目、游戏状态类目、初始NPC
- 三层数据架构：状态层、总结层、设定层
- 永久存储与增量更新
- AI驱动的剧情生成与对话系统

## 三、数据层设计

### 3.1 整体数据结构

```
GameData
├── 状态层 (State Layer)
│   ├── 主角状态 (ProtagonistState)
│   ├── NPC状态 (NPCState[])
│   └── 游戏状态 (GameState)
├── 总结层 (Summary Layer)
│   └── 当前进度总结 (ProgressSummary)
└── 设定层 (Settings Layer)
    ├── 世界观设定 (WorldSetting)
    └── 背景设定 (BackgroundSetting)
```

### 3.2 状态层 (State Layer)

状态层记录游戏的即时数据，随每次交互实时更新。

#### 3.2.1 主角状态 (ProtagonistState)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 唯一标识 |
| name | String | 角色名称 |
| attributes | Map<String, Any> | 动态属性列表 |
| inventory | List<String> | 物品清单 |
| relationships | Map<String, Int> | 与NPC关系值 |
| location | String | 当前位置 |
| createdAt | Long | 创建时间 |
| updatedAt | Long | 更新时间 |

#### 3.2.2 NPC状态 (NPCState)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 唯一标识 |
| name | String | NPC名称 |
| role | String | NPC身份/职业 |
| attributes | Map<String, Any> | 动态属性 |
| dialogueHistory | List<Dialogue> | 对话历史 |
| mood | String | 当前情绪 |
| awareness | String | 对主角的了解程度 |
| updatedAt | Long | 更新时间 |

#### 3.2.3 游戏状态 (GameState)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 唯一标识 |
| currentScene | String | 当前场景 |
| turnCount | Int | 当前轮次 |
| activeEvents | List<String> | 进行中事件 |
| flags | Map<String, Boolean> | 游戏标记 |
| lastAction | String | 最近动作 |
| updatedAt | Long | 更新时间 |

### 3.3 总结层 (Summary Layer)

总结层记录短期进度，每5-10轮对话后由AI自动生成。

#### 3.3.1 进度总结 (ProgressSummary)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 唯一标识 |
| summaryText | String | 总结文本 |
| keyEvents | List<String> | 关键事件列表 |
| involvedNPCs | List<String> | 涉及NPC |
| sceneContext | String | 场景上下文 |
| turnRange | IntRange | 覆盖轮次范围 |
| generatedAt | Long | 生成时间 |

#### 3.3.2 总结层更新机制

```
对话轮次计数器
       │
       ▼
┌──────────────────┐
│ 轮次 % 更新间隔 == 0 │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 调用AI生成新总结   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 替换旧总结层数据   │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│ 作为后续对话上下文 │
└──────────────────┘
```

### 3.4 设定层 (Settings Layer)

设定层记录长期设定，随状态层更新同步变更。

#### 3.4.1 世界观设定 (WorldSetting)

#### 3.4.2 背景设定 (BackgroundSetting)

### 4.2 游戏核心功能

#### 4.2.1 对话系统

```
用户输入 ──► 状态检查 ──► AI生成回复 ──► 状态更新 ──► UI展示
                │                          │
                ▼                          ▼
           回合计数                    触发总结检查
```

#### 4.2.2 状态更新流程

每次对话后的状态更新：

1. 解析AI回复中的状态变更指令
2. 更新主角状态
3. 更新相关NPC状态
4. 更新游戏状态（场景、事件、旗标）
5. 检查是否需要更新总结层
6. 同步更新设定层
7. 持久化存储

#### 4.2.3 存档与读档

- 自动存档：每轮对话后自动保存
- 手动存档：用户可创建命名存档
- 云同步支持（可选）

## 九、关键实现逻辑

### 9.1 总结层生成逻辑

```kotlin
class SummaryManager {
    // 每隔5-10轮生成总结
    fun shouldGenerateSummary(currentTurn: Int, lastSummaryTurn: Int): Boolean {
        val interval = (5..10).random()
        return currentTurn - lastSummaryTurn >= interval
    }

    suspend fun generateSummary(
        recentDialogues: List<Dialogue>,
        currentState: GameState,
        protagonist: Protagonist,
        npcs: List<NPC>
    ): Summary {
        val prompt = buildSummaryPrompt(
            recentDialogues,
            currentState,
            protagonist,
            npcs
        )
        return aiService.generateSummary(prompt)
    }
}
```

### 9.2 设定层同步逻辑

```kotlin
class SettingSyncManager {
    fun syncSettings(stateChange: StateChange, currentSettings: Settings): Settings {
        val updatedSettings = currentSettings.copy()

        // 分析状态变更，提取设定更新
        when (stateChange.type) {
            StateChangeType.NPC_INTRODUCED -> {
                // 新NPC登场，更新背景设定
                updatedSettings.addNPCBackground(
                    stateChange.npcId,
                    "在${stateChange.scene}首次相遇"
                )
            }
            StateChangeType.LORE_UNLOCKED -> {
                // 解锁新 lore
                updatedSettings.unlockedLore.add(stateChange.lore)
            }
            StateChangeType.PLOT_ADVANCED -> {
                // 剧情推进，更新主线
                updatedSettings.majorPlotThreads.add(stateChange.plot)
            }
            // ... 其他类型处理
        }

        return updatedSettings
    }
}
```

### 9.3 对话上下文构建

```kotlin
class ContextBuilder {
    fun buildContext(
        summary: Summary,           // 总结层
        protagonist: Protagonist,    // 状态层-主角
        npcs: List<NPC>,            // 状态层-NPC
        gameState: GameState,       // 状态层-游戏
        worldSetting: WorldSetting,  // 设定层
        background: BackgroundSetting // 设定层
    ): String {
        return buildString {
            appendLine("【世界观设定】")
            appendLine(worldSetting.toPromptString())
            appendLine()
            appendLine("【背景设定】")
            appendLine(background.toPromptString())
            appendLine()
            appendLine("【近期进度】")
            appendLine(summary.summaryText)
            appendLine()
            appendLine("【主角状态】")
            appendLine(protagonist.toPromptString())
            appendLine()
            appendLine("【在场NPC】")
            npcs.forEach { appendLine(it.toPromptString()) }
            appendLine()
            appendLine("【当前场景】")
            appendLine(gameState.currentScene)
        }
    }
}
```

```
┌─────────────────────────────────────────────────────────┐
│                      Presentation                        │
│  (Activities, ViewModels, Composables)                  │
├─────────────────────────────────────────────────────────┤
│                       Domain                              │
│  (UseCases, Repository Interfaces)                      │
├─────────────────────────────────────────────────────────┤
│                        Data                              │
│  ┌─────────────┬─────────────┬─────────────┐           │
│  │ Room DB     │ AI Service  │ Data Store  │           │
│  └─────────────┴─────────────┴─────────────┘           │
└─────────────────────────────────────────────────────────┘

依赖规则：
- Presentation → Domain → Data
- Domain 不依赖 Presentation
- Data 不依赖 Presentation
```

### 三层架构验证

#### 状态层验证
- 主角状态：支持动态属性、物品栏、关系值、位置
- NPC状态：支持动态属性、对话历史、情绪、认知
- 游戏状态：支持场景、轮次、事件、旗标

#### 总结层验证
- 每5-10轮自动触发总结生成
- 总结包含关键事件、涉及NPC、场景上下文
- 总结作为后续对话的上下文基础

#### 设定层验证
- 世界观设定：世界名称、类型、规则、历史、势力、地点
- 背景设定：主角背景、NPC背景、剧情线、已解锁Lore
- 状态变更自动同步更新设定层（追加模式）

### AI接口配置
- 默认使用DeepSeek API
- 接口地址：`https://api.deepseek.com/`
- 模型：`deepseek-chat`
- API Key配置位置：[MainActivity.kt](file:///workspace/textgame/app/src/main/java/com/textgame/MainActivity.kt#L24-L27) 的 `AppModule.configureAI()` 调用处

### 13.2 术语表

| 术语 | 说明 |
|------|------|
| 状态层 | State Layer，存储即时游戏数据 |
| 总结层 | Summary Layer，存储短期进度总结 |
| 设定层 | Settings Layer，存储长期设定 |
| 轮次 | Turn，单次对话交互 |
| 上下文 | Context，发送给AI的完整游戏信息 |
